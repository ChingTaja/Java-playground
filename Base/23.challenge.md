```java
int myIntValue = 5;
float myFloatValue = 5F;
double myDoubleValue = 5D;
```
輸出結果
int → 5
float → 5.0
double → 5.0

```java
myIntValue = 5 / 2;
```
輸出結果:2
原因:
1. 5 和 2 都是 int
2. Java 判定結果型別為 int , 小數部分直接捨去（不是四捨五入）


- 浮點數除法（float）
```java
myFloatValue = 5F / 2F;
```
輸出結果:2.5


為什麼我們應該選擇 double 呢？
首先，在許多現代電腦上，double 實際上運算得更快
這是因為在硬體晶片層級，電腦本身就具備能更有效率處理
double 數值 的功能，反而比處理對應的 float 還快

第二個原因是，我們在課程後面會接觸到的 Java 函式庫，
尤其是數學相關的函式，
`通常都是設計來處理 double，而不是 float`
並且回傳的結果也都是 double
